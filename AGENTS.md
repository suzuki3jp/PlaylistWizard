# AGENTS.md
このファイルでは AI エージェントが PlaylistWizard を開発するにあたり、知っておくべきことを記述する。

## Project Overview
PlaylistWizard はプレイリストを管理するための Web アプリケーションであり、現在は YouTube, YouTube Music を正式にサポートしている。

## System Architecture
[System Architecture](/assets/system-architecture.excalidraw) には Excalidraw の JSON 形式で、システム全体のアーキテクチャが記述されている。

基本的には [Next.js on Vercel](/apps/www) がユーザーインターフェースであり、そこから [API on Cloudflare stack](/apps/api) にリクエストが送られる。

## ADR
技術的な決定は ADR 形式で、 /docs/adr に保存されている。
ADR を参照、変更する場合は grill-with-docs スキルを使うこと。

## CONTEXT.md
- AI エージェントはユーザーとの会話中に混乱を避けるために、常に厳しく定義されたドメイン用語を使うべきである。
- [CONTEXT.md](/CONTEXT.md) にはすでに定義されたドメイン用語が記述されているため、会話にはそれを使うこと。
- CONTEXT.md を参照、変更したい場合は grill-with-docs スキルを使うこと

## コード中のコメント
- AI エージェントはコードを変更する際、常に適切なコメントを記述するべきである。
- コメントでは、何をするのか、なぜするのかを重点的に書くべきである。
- 関数に対するドキュメントコメントは関数自体が極端に短い場合など、明らかに自明である場合以外は常にあるべきである。
- 意図してベストプラクティスから外れるコードやワークアラウンドを活用する場合は、ドキュメントや意思決定の根拠などをコメントに書くべきで、コンテキストを失わないようにするべきである。
- AI エージェントが後からコードを読む場合理解しやすいように、コードに関するコンテキストは常にコード付近から参照可能である状態を維持すべきである。つまり、ADR を書いた場合、そのドキュメントへのリンクが該当コード箇所に含まれていることが望ましい。
