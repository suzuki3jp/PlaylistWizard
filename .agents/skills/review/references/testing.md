# Testing

# 概要
このアプリケーションでは、以下のテストライブラリが導入されています
- vitest
- @testing-library/react

# テストの実行方法
```bash
pnpm test
```
# テストするかどうかの判断基準
以下のような場合には、テストを書くべきです。

1. 対象のコードが複雑である場合
2. 対象のコードにバグが含まれていた時の影響が大きい場合
3. 対象のコードが純粋であり、単純なユニットテストが書きやすい場合

# テストを書く際のポイント
- モックは最低限にする
- テストの対象はできるだけ小さくする
- テストの名前は英語にする
- 正常系から書き、異常系も書く

# いいテストとは
- テストの失敗が、コードのどこに問題があるのかを明確に示している
- 安定している（flaky ではない）

# テストの重要度
1. 純粋な関数のユニットテスト
2. React コンポーネントのユニットテスト
3. エンドツーエンドテスト
